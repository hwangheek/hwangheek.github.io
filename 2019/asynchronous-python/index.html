2. Generator

мҡ°лҰ¬к°Җ нқ”нһҲ м•Ңкі мһҲлҠ” Pythonмқҳ н•ЁмҲҳ, мҰү def sth_familiar_with(): ...кіј к°ҷмқҙ м •мқҳлҗң н•ЁмҲҳлҘј нҳём¶ңн•  кІҪмҡ°, н•ҙлӢ№ н•ЁмҲҳмқҳ мӢңмһ‘л¶Җн„° лҒқ(нҳ№мқҖ returnмқ„ л§ҢлӮ л•Ң)к№Ңм§Җ м§„н–үмқҙ лҗ©лӢҲлӢӨ. к·ё л’Өм—җ stack frameмқҖ мҶҢл©ёлҗҳлҜҖлЎң, мқҙ н•ЁмҲҳлҘј лӢӨмӢң нҳём¶ңн•  кІҪмҡ° н•ЁмҲҳлҠ” мІҳмқҢл¶Җн„° лӢӨмӢң мӢӨн–үмқҙ лҗ©лӢҲлӢӨ.

GeneratorлҠ” м•Ҫк°„ лӢӨлҰ…лӢҲлӢӨ. кІ°кіјл¬јмқ„ к·ёл•Ңк·ёл•Ң мғқм„ұн•ҙлӮҙлҠ”, мқјмў…мқҳ on-demand function м •лҸ„лЎң мғқк°Ғн•ҙліј мҲҳ мһҲкІ мҠөлӢҲлӢӨ. лӢӨмқҢмқҳ мҳҲмӢңлҘј ліҙкІ мҠөлӢҲлӢӨ.

мң„мҷҖ к°ҷмқҙ н•ЁмҲҳлҘј м •мқҳн•  кІҪмҡ°, CPython м»ҙнҢҢмқјлҹ¬к°Җ н•ЁмҲҳ лӮҙ yield нӮӨмӣҢл“ңлҘј л°ңкІ¬н•©лӢҲлӢӨ. к·ёлҰ¬кі , н•ҙлӢ№ н•ЁмҲҳлҘј generator functionмңјлЎң мІҳлҰ¬н•©лӢҲлӢӨ.

мң„мқҳ 7лІҲ лқјмқёкіј к°ҷмқҙ, generator н•ЁмҲҳк°Җ нҳём¶ңлҗҳл©ҙ generator objectк°Җ мғқм„ұлҗ©лӢҲлӢӨ. 8лІҲ лқјмқём—җм„ң next(gen)мқ„ мӢӨн–үн•ҳл©ҙ, hello_twiceмқҳ 3лІҲ лқјмқёк№Ңм§Җ мӢӨн–үмқҙ лҗ©лӢҲлӢӨ. мӢӨн–ү лҸ„мӨ‘ yield нӮӨмӣҢл“ңлҘј л§ҢлӮ¬мқ„ кІҪмҡ°, н•ҙлӢ№ н•ЁмҲҳмқҳ мӢӨн–үмқҖ мқјмӢңм ҒмңјлЎң ліҙлҘҳлҗң нӣ„ к°’мқ„ return н•©лӢҲлӢӨ. лӢӨмӢң next(gen)мқ„ мӢӨн–үн•  кІҪмҡ° (лқјмқё 12), мқјмӢң ліҙлҘҳлҗҳм—ҲлҚҳ н•ЁмҲҳмқҳ м§„н–үмқҙ, лӢӨмқҢ yieldл¬ёмқ„ л§ҢлӮ л•Ңк№Ңм§Җ кі„мҶҚлҗ©лӢҲлӢӨ. (лқјмқё 4~5) мқҙ кіјм •мқҙ н•ЁмҲҳмқҳ лҒқм—җ лҸ„лӢ¬н•  л•Ң к№Ңм§Җ л°ҳліөлҗ©лӢҲлӢӨ.

Pythonмқҳ stack frameмқҖ (мқҙлҰ„кіј лӢӨлҘҙкІҢ) л©”лӘЁлҰ¬мқҳ stack мҳҒм—ӯм—җ мҳ¬лқјк°ҖлҠ” кІғмқҙ м•„лӢҲлқј, heap мҳҒм—ӯм—җ н• лӢ№лҗ©лӢҲлӢӨ. мқҙлҹ¬н•ң Pythonмқҳ нҠ№м§• лҚ•л¶„м—җ, generatorк°Җ мӢӨн–үмқ„ мқјмӢңм •м§Җн•ҳкі  к°’мқ„ returnн•  л•Ңм—җ stack frameмқ„ мң м§Җн•  мҲҳ мһҲмҠөлӢҲлӢӨ. н•ЁмҲҳк°Җ yieldлҘј л§ҢлӮҳлҠ” мҲңк°„, н•ҙлӢ№ stack frameкіј мҪ”л“ңлҘј м–ҙл””м„ңл¶Җн„° мһ¬к°ңн• м§Җм—җ лҢҖн•ң м •ліҙ(last instruction pointer)лҘј ліҙкҙҖн•ҳлҠ” кІғмһ…лӢҲлӢӨ. (Syed Komail Abbas)

3. Coroutine

мқјлӢЁ, вҖҳCoroutineвҖҷмқҙлқјлҠ” мҡ©м–ҙк°Җ Python-specific keywordк°Җ м•„лӢҲлқјлҠ” мӮ¬мӢӨмқ„ мқём§Җн•  н•„мҡ”к°Җ мһҲмқ„ кІғ к°ҷмҠөлӢҲлӢӨ.

Coroutines are computer-program components that generalize subroutines for non-preemptive multitasking, by allowing multiple entry points for suspending and resuming execution at certain locations.

л”°лқјм„ң, м§Ҳл¬ёмқ„ вҖңPythonмқҳ coroutineмқҖ л¬ҙм—Үмқёк°Җ?вҖқлЎң мһЎлҠ” кІғ ліҙлӢӨлҠ”, вҖңPythonм—җм„ңлҠ” coroutineмқ„ м–ҙл–»кІҢ кө¬нҳ„н•ҳмҳҖлӮҳ?вҖқлЎң мғқк°Ғн•ҳлҠ” мӘҪмқҙ нҳјм„ мқ„ мӨ„мқҙлҠ” л°©н–Ҙмқҙлқјкі  мғқк°Ғн•©лӢҲлӢӨ.

3-1. Generator based вҖҳCoroutineвҖҷ

Pythonмқҙ к°Җм§Җкі  мһҲлҠ” generatorлҘј мқҙмҡ©н•ңлӢӨл©ҙ, coroutineмқ„ кө¬нҳ„н•  мҲҳ мһҲм–ҙ ліҙмһ…лӢҲлӢӨ. н•ЁмҲҳ мӢӨн–үмқ„ мқјмӢңм •м§Җн•ҳкұ°лӮҳ мһ¬к°ңн•  мҲҳ мһҲлҠ” entry pointл“Өмқҙ мЎҙмһ¬н•ңлӢӨлҠ” м җкіј, к·ё л•Ңл§ҲлӢӨ к°’мқ„ yield н•  мҲҳ мһҲлҠ” generatorмқҳ м„ұм§Ҳмқ„ мһҳ мқҙмҡ©н•ҳлҠ” кІғмқҙм§Җмҡ”.

Python generatorмҷҖ кҙҖл Ён•ң л‘җк°Җм§Җ кё°лҠҘмқ„ лҚ” мӮҙнҺҙлҙ…мӢңлӢӨ. н•ҳлӮҳлҠ” yieldлҗҳм–ҙ мӢӨн–үмқҙ мқјмӢңм •м§Җ лҗҳм–ҙмһҲлҠ” generatorм—җкІҢ к°’мқ„ м „лӢ¬н•ҳлҠ” кё°лҠҘмһ…лӢҲлӢӨ. м•„лһҳмқҳ мҳҲмӢңлҘј ліҙкІ мҠөлӢҲлӢӨ.

gen.send('Hello')лЎң мқён•ҙ generatorмқҳ мӢӨн–үмқҙ мһ¬к°ң лҗ  кІҪмҡ°, yield msg_first мӢ нғқмҠӨ мһҗмІҙк°Җ 'Hello'лЎң мІҳлҰ¬лҗ©лӢҲлӢӨ. мқҙмІҳлҹј, мӢӨн–үмқҙ л©Ҳм¶°мһҲлҠ” generator objectмқҳ send methodлҘј нҳём¶ңн•ЁмңјлЎңмҚЁ, generatorм—җкІҢ лҚ°мқҙн„°лҘј м „лӢ¬н•  мҲҳ мһҲмҠөлӢҲлӢӨ.

лӢӨлҘё н•ҳлӮҳлҠ”, generatorм—җм„ң лӢӨлҘё generatorлҘј yieldн•ҳлҠ” кё°лҠҘмһ…лӢҲлӢӨ. мқҙлҠ” PEP380 (Syntax for Delegating to a Subgenerator)м—җ лӘ…мӢңлҗҳм–ҙмһҲмҠөлӢҲлӢӨ. м•„лһҳмқҳ мҳҲмӢңлҘј ліҙкІ мҠөлӢҲлӢӨ.

лқјмқё 8кіј к°ҷмқҙ yield from нӮӨмӣҢл“ңлҘј мӮ¬мҡ©н•ҳм—¬ generator лӮҙл¶Җм—җм„ңлҸ„ лӢӨлҘё generatorлҘј мғқм„ұн•ҳм—¬ yieldлҘј мӮ¬мҡ©н•  мҲҳ мһҲмҠөлӢҲлӢӨ. мҰү, yield from inner()лҠ” for x from inner(): yield xмҷҖ к°ҷмқҖ м—ӯн• мқ„ н•ҳлҠ” м…Ҳмһ…лӢҲлӢӨ. PEP380м—җ л”°лҘҙл©ҙ, мқҙ мӢ нғқмҠӨлҠ” Python 3.3л¶Җн„° м¶”к°Җлҗҳм—ҲлӢӨкі  н•©лӢҲлӢӨ. (л”°лқјм„ң, Python 2.7м—җм„ң мң„мқҳ мҳҲмӢңлҘј мӢӨн–үн•ҳл©ҙ yield from мӢ нғқмҠӨлҘј мқҙн•ҙн•ҳм§Җ лӘ»н•ҳкі  м—җлҹ¬к°Җ л°ңмғқн•©лӢҲлӢӨ.)

мң„м—җм„ң м •мқҳлҗң echo_meмҷҖ inner, outerлҠ” мқјмў…мқҳ coroutineмһ…лӢҲлӢӨ. (мӢӨн–үмқ„ мқјмӢңм •м§Җн•ҳкі  мһ¬к°ңн•  мҲҳ мһҲлҠ” entry pointsл“Өмқҙ мЎҙмһ¬н•ҳкё° л•Ңл¬ёмһ…лӢҲлӢӨ.) GeneratorлҘј мқҙмҡ©н•ҳм—¬ л§Ңл“Өм—ҲмңјлҜҖлЎң, вҖҳgenerator based coroutineвҖҷлқјкі  мғқк°Ғн•  мҲҳ мһҲкІ мҠөлӢҲлӢӨ. (м—¬кё°к№Ңм§ҖлҠ” generatorмҷҖ coroutineмқҙлқјлҠ” мҡ©м–ҙмқҳ кө¬л¶„мқҙ лӘ…нҷ•н•ҳм§Җ м•ҠмҠөлӢҲлӢӨ.)

мқҙлҹ¬н•ң generator based coroutineл“Өмқҙ к°’мқ„ yield н•  кІҪмҡ°, н•ӯмғҒ мһҗмӢ мқҳ callerм—җкІҢлЎң мӢӨн–үмқҳ мҲңм„ңк°Җ л„ҳм–ҙк°ҖкІҢ лҗ©лӢҲлӢӨ. мқҙлҹ¬н•ң coroutineмқ„ вҖҳAsymmetric coroutineвҖҷмқҙлқјкі  л¶ҖлҘҙкё°лҸ„ н•©лӢҲлӢӨ. мқҙмҷҖ кө¬л¶„лҗҳлҠ” к°ңл…җмқё вҖҳSymmetric coroutineвҖҷмқҳ кІҪмҡ°м—җлҠ” н•ҳлӮҳмқҳ coroutineм—җм„ң к°’мқ„ yieldн•  л•Ң, лӢӨлҘё coroutineм—җкІҢ мӢӨн–ү мҲңм„ңлҘј л„ҳкІЁмӨ„ мҲҳ мһҲлҠ” нҶөм ңк¶Ңмқ„ к°Җм§Җкі  мһҲмҠөлӢҲлӢӨ. л¬ҙмЎ°кұҙ мһҗмӢ мқҳ callerм—җкІҢ мӢӨн–ү мҲңм„ңлҘј л„ҳкІЁмЈјм–ҙм•ј н•ҳлҠ” asymmetric coroutineкіјлҠ” лӢӨлҘҙмЈ . (Moura)

3-2. asyncio Module

л¬ҙмЎ°кұҙ callerмҷҖл§Ң мғҒнҳёмһ‘мҡ©н•ҳлҠ” л°ҳмӘҪм§ңлҰ¬ coroutineмқҙлқјлӢҲвҖҰ. л§Ңм•Ҫ callerк°Җ мқҙлҹ¬н•ң asymmetric coroutineл“Өмқ„ мң кё°м ҒмңјлЎң м—®м–ҙмЈјлҠ” м—ӯн• мқ„ н•ҙмӨҖлӢӨл©ҙ (мҳҲлҘј л“Өл©ҙ н•ҳлӮҳмқҳ coroutineмқҙ suspend лҗҳм—Ҳмқ„ л•Ң лӢӨлҘё coroutineмқ„ мӢӨн–үмӢңмјңмЈјлҠ” м—ӯн• мқ„ н•ҙмӨҖлӢӨл©ҙ), м „мІҙм ҒмңјлЎң ліҙм•ҳмқ„ л•Ң к°Ғк°Ғмқҳ coroutineл“Өмқҙ м„ңлЎңм—җкІҢ мӢӨн–ү мҲңм„ңлҘј л„ҳкІЁмЈјлҠ” кІғ мІҳлҹј ліҙмқј мҲҳ мһҲкІ мҠөлӢҲлӢӨ.

asyncio лӘЁл“Ҳмқҙ мқҙлҹ¬н•ң м—ӯн• мқ„ н•ҙмӨҚлӢҲлӢӨ. м•„лһҳмқҳ мҳҲмӢңлҘј ліҙкІ мҠөлӢҲлӢӨ. asyncio docsмқҳ мҳҲмӢң мӨ‘ н•ҳлӮҳлҘј м°ёкі н•ҳмҳҖмҠөлӢҲлӢӨ.

asyncio.coroutine лҚ°мҪ”л Ҳмқҙн„°лҘј нҶөн•ҙ echo_twice н•ЁмҲҳлҘј coroutineмңјлЎң м •мқҳн•©лӢҲлӢӨ. asyncio.ensure_future(...)лҘј нҶөн•ҙ, мӢӨн–үн•  coroutineмқ„ мғқм„ұн•©лӢҲлӢӨ. asyncio.get_event_loop()лЎң л°ӣмқҖ event loopмқҳ run_foreverлҘј мӢӨн–үн•ЁмңјлЎңмҚЁ coroutineл“Өмқ„ мӢӨн–үн•©лӢҲлӢӨ.

м¶ңл Ҙлҗң л©”мӢңм§ҖлҘј нҷ•мқён•ҙліҙл©ҙ, л‘җ к°ңмқҳ coroutineмқҙ лҸҷмӢңм—җ м§„н–үлҗң кІғмқ„ нҷ•мқён•  мҲҳ мһҲмҠөлӢҲлӢӨ. echo_twice coroutine мӢӨн–ү мӨ‘ лқјмқё 5м—җ лӢӨлӢӨлҘҙл©ҙ asyncio.sleep(1)мқҙ 1мҙҲ л’Өм—җ мӢӨн–үмқҙ мҷ„лЈҢлҗҳлҠ” (1мҙҲ л’Өм—җ yieldн•  мҲҳ мһҲлҠ”) coroutineмқ„ мғқм„ұн•©лӢҲлӢӨ. мқҙ coroutineмқ„ yield fromн•ҳкІҢ лҗҳл©ҙ event loopм—җкІҢлЎң нҶөм ңк¶Ңмқҙ л„ҳм–ҙк°Җкі , event loopм—җ мқҳн•ҙм„ң лӢӨлҘё coroutineмқҙ мӢӨн–үлҗ©лӢҲлӢӨ. (Mansnun)

мқҙлҹ¬н•ң л°©лІ•мқ„ нҶөн•ҙ, м—¬лҹ¬ coroutineл“Өмқҙ лҸҷмӢңм—җ мӢӨн–үлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ. м—¬кё°м„ң л§җн•ҳлҠ” вҖҳлҸҷмӢңм—җвҖҷ мӢӨн–үлҗңлӢӨлҠ” л§җмқҖ parallellyк°Җ м•„лӢҲлқј concurrentlyлҘј мқҳлҜён•©лӢҲлӢӨ. Event loopмқҖ single threadмқҙлҜҖлЎң н•ңлІҲм—җ мӢӨн–ү лҗҳлҠ” coroutineмқҖ н•ҳлӮҳмһ…лӢҲлӢӨ. м—¬лҹ¬ coroutineл“Өмқҙ н•ҳлӮҳмқҳ threadлҘј нғҖмқҙл°Қ мўӢкІҢ лӮҳлҲ„м–ҙ к°Җм§ҖлҠ” м…Ҳмһ…лӢҲлӢӨ. (Masnun) Concurrencyм—җ кҙҖн•ң мқҙм•јкё°мҷҖ asyncioм—җ кҙҖн•ң мЎ°кёҲ лҚ” л””н…Ңмқјн•ң м„ӨлӘ…мқҖ л’Өм—җм„ң лӢӨмӢң н•ҳкІ мҠөлӢҲлӢӨ.

3-3. Native Coroutine

Python 3.5л¶Җн„° мғҲлЎңмҡҙ мӢ нғқмҠӨмқё asyncмҷҖ awaitк°Җ л“ұмһҘн•©лӢҲлӢӨ. (PEP492) мқҙлҘј мқҙмҡ©н•ҳл©ҙ мң„м—җм„ң м„ м–ён•ң echo_twiceлҘј м•„лһҳмҷҖ к°ҷмқҙ лӢӨмӢң м •мқҳн•  мҲҳ мһҲмҠөлӢҲлӢӨ.

@asyncio.coroutine лҚ°мҪ”л Ҳмқҙн„° лҢҖмӢ  async нӮӨмӣҢл“ңлҘј, yield from нӮӨмӣҢл“ң лҢҖмӢ  await нӮӨмӣҢл“ңлҘј мӮ¬мҡ©н–ҲмҠөлӢҲлӢӨ. Pythonмқҳ мӢ нғқмҠӨл§Ңмқ„ мқҙмҡ©н•ҳм—¬ м •мқҳн•ң coroutineмқҙлӢҲ, мқҙлҘј вҖҳNative CoroutineвҖҷмқҙлқјкі  л¶Ҳлҹ¬лҸ„ кҙңм°®кІ мҠөлӢҲлӢӨ. (Masnun) мӢӨн–ү лҸ„мӨ‘ лқјмқё 4мқҳ await asyncio.sleep(1)м—җ лӢӨлӢӨлҘҙл©ҙ, echo_twice н•ЁмҲҳлҠ” event loopм—җкІҢ м»ЁнҠёлЎӨмқ„ л„ҳкІЁмӨҚлӢҲлӢӨ.
мқҙ л•Ң, generator based coroutineм—җм„ң мӮ¬мҡ©лҗҳлҠ” мӢ нғқмҠӨмҷҖ native coroutineм—җм„ң мӮ¬мҡ©лҗҳлҠ” мӢ нғқмҠӨлҠ” нҳјмҡ©лҗ  мҲҳ м—ҶмҠөлӢҲлӢӨ. л”°лқјм„ң, м•„лһҳмҷҖ к°ҷмқҖ мҪ”л“ңлҠ” н—Ҳмҡ©лҗҳм§Җ м•ҠмҠөлӢҲлӢӨ.

4. ConcurrencyмҷҖ Parallelism

CPythonмқҳ л©”лӘЁлҰ¬ кҙҖлҰ¬лҠ” thread-safeн•ҳм§Җ м•ҠмҠөлӢҲлӢӨ. л”°лқјм„ң GIL(Global Interpreter Lock)мқҙлқј л¶ҲлҰ¬лҠ” mutexлҘј мӮ¬мҡ©н•ҳм—¬ м—¬лҹ¬ м“°л Ҳл“ңк°Җ лҸҷмӢңм—җ Python мҪ”л“ңлҘј мӢӨн–үн•ҳм§Җ лӘ»н•ҳлҸ„лЎқ ліҙнҳён•©лӢҲлӢӨ. (ThomasWouters)

м—¬кё°м„ң лЁёлҰ¬к°Җ ліөмһЎн•ҙмЎҢмҠөлӢҲлӢӨ. к·ёлҹј мң„м—җм„ң л§җн–ҲлҚҳ лҸҷмӢңм—җ мӢӨн–үлҗҳлҠ” coroutineмқҖ м–ҙл–»кІҢ вҖҳлҸҷмӢңм—җвҖҷ лҸҢм•„к°Ҳ мҲҳ мһҲлҠ” кІғмқјк№Ңмҡ”? мқҙлҘј мһҳ мқҙн•ҙн•ҳкё° мң„н•ҙм„ңлҠ” concurrencyмҷҖ parallelismмқ„ кө¬л¶„н•ҳм—¬ м •мқҳлӮҙлҰҙ н•„мҡ”к°Җ мһҲмҠөлӢҲлӢӨ.

Concurrency is about dealing with lots of things at once
Parallelism is about doing lots of things at once

Skrew EverythingDon't be confused between Concurrency and Parallelism

л‘җ мҡ©м–ҙлҠ” лӢӨлҘё м°Ёмӣҗмқҳ к°ңл…җмһ…лӢҲлӢӨ. ParallelismмқҖ н•ҳлӮҳмқҳ мһ‘м—…мқҙ м—¬лҹ¬к°ңмқҳ мһ‘мқҖ мһ‘м—…л“ӨлЎң лӮҳлүҳм–ҙм ё л¬јлҰ¬м ҒмңјлЎң лҸҷмӢңм—җ м§„н–үлҗҳлҠ” кІғмқ„ л§җн•©лӢҲлӢӨ. м—¬лҹ¬к°ңмқҳ CPUк°Җ лҸҷмӢңм—җ лҸҢм•„к°ҖлҠ” кІғмқ„ мғқк°Ғн•ҳл©ҙ лҗҳкІ мҠөлӢҲлӢӨ. л°ҳл©ҙ, ConcurrencyлҠ” м—¬лҹ¬ мһ‘м—…л“Өмқҙ к°ҷмқҖ мҲңк°„м—җ (physically or logically) мІҳлҰ¬лҗҳкі  мһҲлҠ” мғҒнғңмһ…лӢҲлӢӨ.

мқҙ м •мқҳлҘј нҶөн•ҙ м•Ң мҲҳ мһҲлҠ” мӮ¬мӢӨмқҙ лӘҮ к°Җм§Җ мһҲмҠөлӢҲлӢӨ.

  1. Single Thread нҷҳкІҪм—җм„ңлҸ„ concurrentн•ң мһ‘м—…мқ„ мҲҳн–үн•  мҲҳ мһҲлӢӨ. (CPU time-slicing нҳ№мқҖ Application levelмқҳ schedulingмқ„ нҷңмҡ©)
  2. Parallelн•ң мһ‘м—…мқҖ Concurrentн•ҳлӢӨ. к·ё л°ҳлҢҖлҠ” н•ӯмғҒ м„ұлҰҪн•ҳм§Җ м•ҠлҠ”лӢӨ. (Skrew Everything)

Pythonмқҳ coroutineл“ӨмқҖ concurrentн•ҳкІҢ мӢӨн–үлҗ©лӢҲлӢӨ. Parallelн•ҳкІҢ лҸҢм•„к°ҖлҠ” кІғмқҙ м•„лӢҷлӢҲлӢӨ. м•һм„ң мқҙм•јкё° н–Ҳл“Ҝмқҙ, PythonмқҖ GILлЎң мқён•ҳм—¬ н•ңлІҲм—җ н•ҳлӮҳмқҳ threadл§Ң мӢӨн–үмқҙ лҗ©лӢҲлӢӨ. л”°лқјм„ң, мң„м—җм„ң мқҙм•јкё° н–ҲлҚҳ asynchronousн•ң coroutineл“ӨмқҖ вҖҳOne process, One threadвҖҷ нҷҳкІҪм—җм„ң concurrentн•ҳкІҢ лҸҢм•„к°ҖлҠ” кІғмһ…лӢҲлӢӨ. м•„лһҳлҠ” Skrew EverthingлӢҳмқҳ нҸ¬мҠӨнҠём—җ мӮҪмһ…лҗң Parallelismкіј Concurrency without Parallelismмқ„ мӢңк°Ғнҷ”н•ң к·ёлҰјмһ…лӢҲлӢӨ. Pythonмқҳ GILмқ„ кі л Өн•ҳмҳҖмқ„ л•Ң, к·ёлҰјмқҳ м•„лһ«л¶Җл¶„м—җ мһҲлҠ” Concurrency without Parallelismмқҙ Pythonм—җм„ңмқҳ coroutineл“Өмқҳ лҸҷмһ‘ нҳ•нғңмҷҖ мң мӮ¬н•ҳлӢӨкі  ліј мҲҳ мһҲмҠөлӢҲлӢӨ.

к·ёлҹ¬лҜҖлЎң, вҖңн•ҳлӮҳмқҳ мһ‘м—…мқ„ м—¬лҹ¬к°ңмқҳ coroutineмңјлЎң лӮҳлҲ„м–ҙ мӢӨн–ү н•  кІҪмҡ° мһ‘м—…мқҙ л№Ёлқјм§„лӢӨвҖқлқјлҠ” statementлҠ” н•ӯмғҒ мҳім§Җ м•ҠмҠөлӢҲлӢӨ. вҖҳOne process, One threadвҖҷ нҷҳкІҪм—җм„ң лҸҢм•„к°Җкё° л•Ңл¬ём—җ, coroutineмқ„ context switchingн•ҳлҠ” costлЎң мқён•ҳм—¬ кІҪмҡ°м—җ л”°лқјм„ңлҠ” performanceк°Җ лҚ” лӮҳл№ м§Ҳ к°ҖлҠҘм„ұлҸ„ мһҲмҠөлӢҲлӢӨ.

5. More about async, await and asyncio

Awaitable Objects

__await__() methodк°Җ кө¬нҳ„лҗҳм–ҙ мһҲлҠ” objectлҘј awaitable objectлқјкі  н•©лӢҲлӢӨ. (Python Docs) мқҙлҰ„ к·ёлҢҖлЎң, await лҗ  мҲҳ мһҲлҠ” objectлҘј мқҳлҜён•©лӢҲлӢӨ. Coroutines, Tasks, Futuresк°Җ awaitable objectм—җ мҶҚн•©лӢҲлӢӨ.

Coroutines
м•һм„ң м„ӨлӘ…н•ң coroutineл“ӨмқҖ awaitable objectмһ…лӢҲлӢӨ. л”°лқјм„ң, лӢӨлҘё coroutineм—җм„ң await лҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.

Futures
FutureлҠ” л№„лҸҷкё° м—°мӮ°мқҳ кІ°кіјл¬јмқ„ лҢҖмӢ н•ҳлҠ” low-level awaitable objectмһ…лӢҲлӢӨ. (A Future is a special low-level awaitable object that represents an eventual result of an asynchronous operation. / Represents the result of an asynchronous computation.)
Future objectк°Җ awaitлҗ  кІҪмҡ°, лӢӨлҘё кіім—җм„ң resolve лҗҳкё° м „ к№Ңм§Җ н•ҙлӢ№ coroutineмқҙ кё°лӢӨлҰ¬кІҢ лҗ©лӢҲлӢӨ. мқјл°ҳм Ғмқё кІҪмҡ°, application level мҪ”л“ңм—җм„ң Future objectлҘј мғқм„ұ н•  н•„мҡ”лҠ” м—ҶмҠөлӢҲлӢӨ.

Tasks
TaskлҠ” coroutineмқ„ concurrentн•ҳкІҢ мӢӨн–үн•ҳкё° мң„н•ҳм—¬ scheduleмқ„ н•ҙ лҶ“мқҖ objectмһ…лӢҲлӢӨ. asyncio.create_task() н•ЁмҲҳ л“ұмқ„ мқҙмҡ©н•ҳм—¬ coroutineмқ„ TaskлЎң wrapping н•  кІҪмҡ°, н•ҙлӢ№ coroutineмқҖ мһҗлҸҷм ҒмңјлЎң schedule лҗҳм–ҙм„ң мӢӨн–ү к°ҖлҠҘн•ң мғҒнғңк°Җ лҗ©лӢҲлӢӨ. л”°лқјм„ң, мң„мқҳ main_coroutines()м—җм„ңлҠ” main_coroutine()кіј coro()к°Җ concurrentн•ҳкІҢ лҸҢм•„к°Җм§Җ м•Ҡм§Җл§Ң, м•„лһҳмқҳ main_tasks()м—җм„ңлҠ” main_tasks()мҷҖ coro()к°Җ concurrentн•ҳкІҢ мӢӨн–үлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.

asyncio module

asyncioмқҳ method мӨ‘ н•өмӢ¬м Ғмқё лӘҮ к°Җм§ҖлҘј м •лҰ¬н•ҙліҙл©ҙ м•„лһҳмҷҖ к°ҷмҠөлӢҲлӢӨ.

asyncio.run(coro, *, debug=False)

м „лӢ¬ лҗң coroutineмқ„ мӢӨн–үн•ҳкі , asyncioмқҳ event loopлҘј кҙҖлҰ¬н•©лӢҲлӢӨ. к°ҷмқҖ threadм—җм„ң лӢӨлҘё asyncio event loopк°Җ мӢӨн–үмӨ‘мқј кІҪмҡ°м—җлҠ” мӢӨн–үн•  мҲҳ м—ҶмҠөлӢҲлӢӨ.
Python 3.7м—җм„ң мғҲлЎң м¶”к°Җлҗҳм—ҲмҠөлӢҲлӢӨ.

asyncio.create_task(coro)

м „лӢ¬ лҗң coroutineмқ„ wrappingн•ҳм—¬ Task objectлҘј л§Ңл“Өкі  schedule н•©лӢҲлӢӨ.
Python 3.7м—җм„ң мғҲлЎң м¶”к°Җлҗҳм—ҲмҠөлӢҲлӢӨ. мқҙм „ Pythonм—җм„ңлҠ” asyncio.ensure_future()лҘј мӮ¬мҡ©н•©лӢҲлӢӨ.

asyncio.sleep(delay, result=None, loop=None)

delayмҙҲ лҸҷм•Ҳ лҢҖкё°н•©лӢҲлӢӨ. resultк°Җ мЈјм–ҙм§„ кІҪмҡ°, sleepмқҙ мҷ„лЈҢлҗң нӣ„ н•ҙлӢ№ к°’мқ„ callerм—җкІҢ лҰ¬н„ҙн•©лӢҲлӢӨ.
sleep() н•ЁмҲҳлҠ” нҳ„мһ¬ taskлҘј suspend мӢңнӮӨлҜҖлЎң, мқҙ мӢңм җм—җ лӢӨлҘё taskк°Җ мӢӨн–үлҗ  мҲҳ мһҲмҠөлӢҲлӢӨ.

asyncio.gather(*aws, loop=None, return_exceptions=None)

мЈјм–ҙм§„ awaitable objectsл“Өмқ„ concurrentн•ҳкІҢ мӢӨн–үн•©лӢҲлӢӨ. мЈјм–ҙм§„ awaitable object мӨ‘ coroutineмқҙ мһҲмңјл©ҙ, мһҗлҸҷмңјлЎң TaskлЎң schedule лҗ©лӢҲлӢӨ.
лӘЁл“  awaitableл“Өмқҙ м •мғҒм ҒмңјлЎң лҒқлӮҳл©ҙ, к°Ғ awaitableмқҳ return valueмқҳ listк°Җ return лҗ©лӢҲлӢӨ. мҲңм„ңлҠ” awsмқҳ мҲңм„ңмҷҖ лҸҷмқјн•©лӢҲлӢӨ.

asyncio.wait_for(aw, timeout, *, loop=None)

мЈјм–ҙм§„ awaitableмқ„ timeoutмҙҲ лҸҷм•Ҳ кё°лӢӨлҰҪлӢҲлӢӨ. awк°Җ coroutineмқј кІҪмҡ°, мһҗлҸҷмңјлЎң TaskлЎң schedule лҗ©лӢҲлӢӨ.
н•ҙлӢ№ мӢңк°„мқҙ м§ҖлӮҳл©ҙ, asyncio.TimeoutErrorк°Җ л°ңмғқн•©лӢҲлӢӨ.

вҖҰ

м¶”к°Җм Ғмқё м„ӨлӘ…мқҖ Python Docмқ„ м°ёмЎ°н•ҳл©ҙ лҗҳкІ мҠөлӢҲлӢӨ.

Using asyncio based module : aiohttp

aiohttpлҠ” asyncioлҘј нҷңмҡ©н•ң HTTP нҒҙлқјмқҙм–ёнҠё/м„ңлІ„ лӘЁл“Ҳмһ…лӢҲлӢӨ. мқҙ лӘЁл“Ҳкіј мң„мқҳ лӮҙмҡ©л“Өмқ„ л°”нғ•мңјлЎң лӢӨмҲҳмқҳ мӣ№мӮ¬мқҙнҠёлҘј л¶Ҳлҹ¬мҳӨлҠ” asynchronousлҘј л§Ңл“Өм–ҙліҙл©ҙ лӢӨмқҢкіј к°ҷмҠөлӢҲлӢӨ.

мқҙмҷҖ к°ҷмқҖ лҸҷмһ‘мқ„ н•ҳлҠ” synchronousн•ң мҠӨнҒ¬лҰҪнҠёлҠ” мӢңк°„мқҙ м–јл§ҲлӮҳ кұёлҰҙк№Ңмҡ”? м•„лһҳмҷҖ к°ҷмқҙ к°„лӢЁн•ң н…ҢмҠӨнҠёлҘј н•ҙліҙм•ҳмҠөлӢҲлӢӨ.
